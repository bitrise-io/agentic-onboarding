How it works

Three skills, one seamless journey

Each skill is a Claude Code slash command that drives a stage of the onboarding. Run them individually or let the orchestrator chain all three automatically.

1

Create your Bitrise account

bitrise-signup

Claude registers a new account via anonymous MCP tools, handles OTP verification, and wires your fresh PAT into the MCP config — no browser login required.

Say to Claude
"Sign me up to Bitrise"
"Create my Bitrise account"
"I want to try Bitrise"
Step details
  1. Claude asks which email to register.
  2. Calls register — a 6-digit OTP is emailed to you.
  3. You paste the OTP; Claude calls verify_registration.
  4. Runs claude mcp remove bitrise then re-adds it with -e BITRISE_TOKEN=… — the PAT is never echoed in chat.
  5. After you restart Claude, calls me + list_workspaces to confirm everything works.
2

Register your app & run your first build

bitrise-ci-onboarding

Claude registers your repo, generates and uploads an SSH deploy key, picks the right stack, uploads signing credentials, triggers the first build, and reads the logs on failure.

Say to Claude
"Set up my project on Bitrise"
"Run my first Bitrise build"
"Get my iOS/Android app building on Bitrise"
Step details
  1. Claude collects workspace, platform (iOS / Android), repo URL, and git provider.
  2. Calls register_app to create the project.
  3. For private repos: generates an ed25519 key and calls register_ssh_key to auto-add it as a GitHub/GitLab deploy key.
  4. Calls list_available_stacks, picks the right stack, calls finish_bitrise_app.
  5. Calls register_webhook so git pushes auto-trigger builds.
  6. Uploads signing files via upload_build_certificate + upload_provisioning_profile (iOS) or upload_android_keystore_file (Android).
  7. Calls trigger_bitrise_build and shares the build URL immediately.
  8. Polls get_build every 15–30 s; calls get_build_log on failure to diagnose.
3

Share your build with testers

bitrise-release-management

Claude creates a Release Management connected app, uploads the IPA / APK / AAB via presigned URL, enables the public install page, and hands you back a shareable link.

Say to Claude
"Share my build"
"Get me a public install link"
"Set up Release Management"
Step details
  1. Claude confirms workspace, platform, bundle ID / package name, and artifact source.
  2. Calls create_connected_app with manual_connection: true.
  3. If the artifact is a CI build slug: fetches it via list_artifacts → get_artifact → downloads with curl.
  4. Generates a client-side UUID and calls generate_installable_artifact_upload_url with with_public_page: true.
  5. Uploads via curl -X PUT using the presigned URL and headers.
  6. Polls get_installable_artifact_upload_and_processing_status until ready.
  7. Calls set_installable_artifact_public_install_page as a defensive confirmation.
  8. Calls list_installable_artifacts and surfaces the public install page URL.

Run the full journey in one shot

The bitrise-onboard-end-to-end orchestrator chains all three skills with shared state — one conversation, one context, ~15–30 minutes from zero to public install link.

Trigger phrases
"Onboard me to Bitrise from scratch" "Give me the full Bitrise demo" "Go from zero to a shareable install link"

Estimated total time: 15–30 minutes, dominated by the first build and OTP email latency.

Installation

Two commands. No credentials needed upfront.

Requires Claude Code CLI and Go ≥ 1.25. The plugin auto-wires the Bitrise MCP — no separate server setup needed.

1

Add the marketplace catalog

claude plugin marketplace add bitrise-io/agentic-onboarding

Registers the Bitrise agentic-onboarding plugin catalog from GitHub into your local Claude Code installation.

2

Install the plugin

claude plugin install bitrise-agentic-onboarding@agentic-onboarding

Installs the plugin and auto-wires the Bitrise MCP via go run on the agentic-onboarding branch. First launch compiles the MCP binary — allow 30–90 s of silence.

3

Launch Claude and start

claude

# Then say:
Onboard me to Bitrise from scratch.

The orchestrator skill takes over — it will ask for your email, then your app details. Nothing else to configure in advance.

Update & uninstall 
# Update to the latest version
claude plugin marketplace update agentic-onboarding

# Uninstall
claude plugin uninstall bitrise-agentic-onboarding@agentic-onboarding
claude plugin marketplace remove agentic-onboarding
Requirements

What you need before you start

🐹

Go ≥ 1.25

The MCP server is a Go binary compiled on first launch. Check with go version or install from go.dev/dl.

🖥️

Claude Code CLI

Requires the claude CLI. The Cowork desktop app's local plugin install isn't supported yet — use the CLI for now.

🌐

Internet on first run

Go pulls the agentic-onboarding branch from proxy.golang.org on first launch. Subsequent starts use the local module cache.
